This PR fixes the Render deployment failure caused by the missing start script at the repository root.

## Problem
Render executes yarn start from the repository root. The root package.json did not have a start script, causing:

error Command "start" not found.

## Changes
- root package.json: Added start script pointing to backend
- root Procfile: Added web start command for Render
- root render.yaml: Added Render Blueprint with correct build/start commands
- RENDER_DEPLOYMENT_GUIDE.md: Updated instructions for root-level deployment

## Deployment
With these changes, Render can deploy directly from the repository root without requiring backend to be set as the Root Directory.
